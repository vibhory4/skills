# Twenty-One Execution Tactics

Use this reference to select the smallest intervention that addresses the user's actual friction. Combine tactics only when they solve different parts of the problem.

## 1. Define the target (Set the Table)

**Principle:** Clarity reduces avoidance.

**Use when:** The task is vague, the goal is unwritten, or expectations are uncertain.

**Procedure:**

1. State the outcome in observable terms.
2. Confirm what good enough means and who accepts the result.
3. Set a real deadline and useful intermediate checkpoints.
4. List the required actions and order dependencies.
5. Begin one action immediately and identify a daily progress action for longer goals.

**Output:** Written goal, done condition, deadline, ordered actions, and first move.

**Failure mode:** Polishing a plan without starting it.

## 2. Plan before reacting (Plan Every Day in Advance)

**Principle:** A short planning pass prevents the day from being controlled by arrivals and interruptions.

**Use when:** The user starts with inboxes, forgets commitments, or changes direction repeatedly.

**Procedure:**

1. Maintain a master capture list.
2. Translate it into monthly, weekly, and daily horizons as needed.
3. Build tomorrow's list before the current day ends.
4. Order tasks by both priority and dependency.
5. Carry forward unfinished work deliberately rather than automatically.

**Output:** A realistic daily sequence tied to weekly outcomes.

**Failure mode:** Filling every minute and leaving no buffer for real work or disruption.

## 3. Find the vital few (Apply the 80/20 Rule)

**Principle:** A small number of activities usually create most of the meaningful result.

**Use when:** The user is busy but progress on important outcomes is weak.

**Procedure:**

1. List current tasks, customers, projects, or deliverables.
2. Identify the few that drive the largest outcome or prevent the largest loss.
3. Start one of those before clearing small, easy tasks.
4. Reduce time spent on the low-yield majority.

**Output:** A vital-few list and a reduced low-value list.

**Failure mode:** Treating 80/20 as an exact calculation when it is a prioritization heuristic.

## 4. Prioritize by consequences (Consider the Consequences)

**Principle:** Importance is better measured by future consequences than by present discomfort.

**Use when:** Urgent-looking work crowds out work with larger long-term impact.

**Procedure:**

1. Ask what happens if each task is completed well, delayed, or omitted.
2. Consider both near-term and long-term effects.
3. Identify work only the user can do that would materially change the outcome.
4. Choose the most valuable use of the current block.
5. Add a buffer to meaningful deadlines instead of relying on last-minute pressure.

**Output:** A consequence-based priority with a defensible rationale.

**Failure mode:** Using a distant benefit to ignore a genuine immediate obligation.

## 5. Delay low-value work deliberately (Practice Creative Procrastination)

**Principle:** Since not everything can be done, decide consciously what will wait or disappear.

**Use when:** The workload exceeds capacity or new requests continually displace important work.

**Procedure:**

1. Create a not-do or not-now list.
2. Eliminate obsolete work.
3. Delegate work that does not require the user's judgment.
4. Defer low-consequence work until higher-value outcomes are protected.
5. Decline new work when accepting it would displace a more important commitment.

**Output:** Eliminate, delegate, defer, and decline decisions.

**Failure mode:** Quietly ignoring work without communicating with affected people.

## 6. Classify with ABCDE (Use the ABCDE Method)

**Principle:** Force distinctions between must-do, should-do, optional, delegated, and eliminated work.

**Use when:** A task list has many items with no defensible ordering.

**Procedure:**

1. Mark `A` for serious consequences if done or missed.
2. Mark `B` for meaningful but mild consequences.
3. Mark `C` for no material consequence.
4. Mark `D` for work that should be delegated.
5. Mark `E` for work that should be eliminated.
6. Rank multiple A items as A1, A2, and so on.
7. Begin A1 before working on B or C items.

**Output:** A classified list with one A1 task.

**Failure mode:** Labeling everything A and avoiding the required tradeoff.

## 7. Strengthen key result areas (Focus on Key Result Areas)

**Principle:** Performance depends on a small set of results for which the user is accountable.

**Use when:** The user is unclear why their role matters, what to improve, or why effort is not translating into results.

**Procedure:**

1. Identify five to seven outputs that define success in the role.
2. Confirm them with the relevant stakeholder when possible.
3. Rate current performance in each area from 1 to 10.
4. Find the weakest area that limits the value of the others.
5. Create one improvement action and one measure for it.

**Output:** Key result map, weakest-link diagnosis, and improvement action.

**Failure mode:** Listing activities instead of accountable results.

## 8. Concentrate on three contributions (Apply the Law of Three)

**Principle:** Three core activities often account for most of a person's contribution.

**Use when:** A role contains many responsibilities or work is consuming personal time without proportionate value.

**Procedure:**

1. Ask which one activity would create the most value if done all day.
2. Repeat to identify the second and third activities.
3. Validate the three with the user's goals or manager.
4. Reduce, delegate, or standardize supporting work.
5. Protect nonwork priorities rather than allowing efficiency gains to become endless work.

**Output:** Three core contributions and a workload-reduction plan.

**Failure mode:** Declaring support work worthless when it is necessary for delivery or trust.

## 9. Prepare the launchpad (Prepare Thoroughly Before You Begin)

**Principle:** Preparation lowers the effort required to start and continue.

**Use when:** The user repeatedly stops to find files, obtain access, or reorganize the workspace.

**Procedure:**

1. Clear unrelated physical and digital material.
2. Gather inputs, credentials, tools, and references.
3. Arrange them in the order of use.
4. Define an acceptable first version rather than waiting for perfection.
5. Start as soon as preparation is sufficient.

**Output:** A ready workspace and immediate launch action.

**Failure mode:** Turning preparation and organization into a substitute for delivery.

## 10. Navigate one marker at a time (Take It One Oil Barrel at a Time)

**Principle:** Large projects become manageable when attention stays on the next visible milestone.

**Use when:** The final outcome feels too distant or uncertain to begin.

**Procedure:**

1. Sketch the known path to the goal.
2. Identify the next milestone that is visible from the current position.
3. Complete the next action required to reach it.
4. Reassess only after reaching that marker.

**Output:** One near milestone and its next action.

**Failure mode:** Demanding certainty about the entire path before taking a reversible first step.

## 11. Close the skill gap (Upgrade Your Key Skills)

**Principle:** Avoidance often comes from low confidence or competence in one critical skill.

**Use when:** The same type of task is repeatedly slow, stressful, or avoided.

**Procedure:**

1. Name the skill that would make the task easier or faster.
2. Separate learning required now from learning that can occur later.
3. Choose a small practice, course, mentor, or reference.
4. Schedule deliberate practice tied to real work.
5. Track improvement through output quality, speed, or error rate.

**Output:** A focused learning plan connected to a current result.

**Failure mode:** Studying broadly instead of shipping with the minimum skill required now.

## 12. Attack the constraint (Identify Your Key Constraints)

**Principle:** One bottleneck usually controls the speed of the larger outcome.

**Use when:** Effort is high but throughput remains low, or a project repeatedly stalls at the same stage.

**Procedure:**

1. Map the steps from current state to goal.
2. Identify where work waits, fails, or accumulates.
3. Test whether the constraint is internal, external, or misdiagnosed.
4. Focus the next improvement on that limiting factor.
5. Reassess after it moves because a new constraint may appear.

**Output:** Constraint statement, evidence, and one relief experiment.

**Failure mode:** Solving the easiest visible problem rather than validating the real bottleneck.

## 13. Create constructive pressure (Put the Pressure on Yourself)

**Principle:** Self-imposed structure can replace missing supervision or weak urgency.

**Use when:** The task matters but has no immediate external deadline or accountability.

**Procedure:**

1. Set a realistic completion deadline and intermediate checkpoints.
2. Estimate each phase and add a buffer.
3. Create a visible commitment, review, or accountability point.
4. Race the plan only when quality and safety will not suffer.
5. Review misses as planning data, not moral failure.

**Output:** Deadline, checkpoints, and accountability mechanism.

**Failure mode:** Creating arbitrary pressure that causes anxiety, shortcuts, or poor quality.

## 14. Reframe toward action (Motivate Yourself into Action)

**Principle:** Constructive interpretation supports persistence after discomfort or setbacks.

**Use when:** Negative self-talk, blame, or a setback is preventing the next useful action.

**Procedure:**

1. Acknowledge the fact without exaggeration or forced positivity.
2. Separate controllable from uncontrollable factors.
3. Identify one lesson, one available option, and one next step.
4. Restate the goal and evidence of progress.
5. Begin the next controllable action.

**Output:** Balanced reframe and immediate next step.

**Failure mode:** Denying real constraints or emotions with empty affirmations.

## 15. Establish digital boundaries (Technology Is a Terrible Master)

**Principle:** Constant connectivity converts other people's arrivals into the user's agenda.

**Use when:** Messages, news, or social feeds fragment the day or displace important work.

**Procedure:**

1. Disable nonessential notifications.
2. Unsubscribe, filter, or archive low-value inputs.
3. Batch communication into defined windows.
4. Create one or more technology-free focus periods.
5. Preserve an emergency route appropriate to the user's responsibilities.

**Output:** Communication windows, silent zones, and an emergency channel.

**Failure mode:** Disconnecting in a way that violates real response obligations.

## 16. Configure technology as a servant (Technology Is a Wonderful Servant)

**Principle:** Tools should protect priorities and reduce friction rather than create more inputs.

**Use when:** The user needs reliable reminders, delegation, calendar protection, or focused digital workspaces.

**Procedure:**

1. Keep only task-relevant applications and windows open.
2. Route priority contacts to a narrow channel or folder.
3. Protect important work as calendar blocks.
4. Use task software for reminders, delegation, and follow-up.
5. Add a tool only when it removes a recurring friction point.

**Output:** A minimal tool configuration tied to the selected frog.

**Failure mode:** Installing and configuring tools instead of doing the task.

## 17. Defend attention (Focus Your Attention)

**Principle:** Switching attention repeatedly degrades depth, speed, and accuracy.

**Use when:** The user multitasks, checks communication while working, or cannot sustain concentration.

**Procedure:**

1. Choose one task and one block outcome.
2. Remove visible and audible interruption cues.
3. Batch email and messages outside the block.
4. Work for a suitable focused interval, then take a deliberate break.
5. Give meetings and people full attention when they are the chosen task.

**Output:** A distraction-free block with a specific deliverable.

**Failure mode:** Applying long focus intervals when fatigue or task type calls for shorter cycles.

## 18. Slice or timebox the task (Slice and Dice the Task)

**Principle:** Starting a small piece creates momentum and makes an intimidating task concrete.

**Use when:** A project feels too large, unpleasant, or emotionally heavy.

**Procedure:**

Choose one method:

- **Slice:** Break the work into small sequential deliverables and complete one.
- **Timebox:** Work for 5 to 15 minutes on a defined part, then consciously choose whether to continue.

End each slice with a visible artifact or a clearly recorded next action.

**Output:** One completed slice or one short timed start.

**Failure mode:** Making slices so tiny that coordination costs exceed progress.

## 19. Reserve meaningful time (Create Large Chunks of Time)

**Principle:** Important work requires protected calendar capacity, not leftover minutes.

**Use when:** Deep work is continuously postponed or scattered across short gaps.

**Procedure:**

1. Estimate the minimum useful uninterrupted duration.
2. Reserve 30-, 60-, or 90-minute blocks in advance.
3. Assign a concrete output to each block.
4. Batch shallow work around those blocks.
5. Use transition time for small supporting actions, not for work that requires depth.

**Output:** Protected time blocks and deliverables.

**Failure mode:** Reserving large blocks without deciding what will be produced.

## 20. Build momentum quickly (Develop a Sense of Urgency)

**Principle:** After enough planning, prompt action generates information, energy, and momentum.

**Use when:** The next action is known but the user keeps waiting for motivation or ideal conditions.

**Procedure:**

1. Confirm the next step is safe and reversible enough to begin.
2. Start within a short countdown or defined start time.
3. Maintain a steady pace until the first visible result.
4. Report or record progress promptly.
5. Use momentum to continue, not to add unrelated tasks.

**Output:** Immediate action and a visible first result.

**Failure mode:** Confusing speed with importance or bypassing necessary thinking.

## 21. Finish one task (Single Handle Every Task)

**Principle:** Concentrated continuation reduces restart costs and increases completion.

**Use when:** The user starts many tasks, switches frequently, or leaves important work at 80 percent.

**Procedure:**

1. Define what complete means for the current block.
2. Work on only that task.
3. Capture unrelated thoughts without acting on them.
4. If interrupted, record the exact resume point.
5. Continue until completion or the declared checkpoint, then close the loop.

**Output:** A completed deliverable or a clean checkpoint with a resume instruction.

**Failure mode:** Refusing to stop when new information changes the priority or continuing is unsafe.
