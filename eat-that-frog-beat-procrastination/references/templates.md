# Execution Templates

Load only the template needed for the current request. Remove unused fields rather than returning an empty form.

## Quick start

```markdown
Frog: [task]
Done when: [observable condition]
Start now: [action that takes no more than 10 minutes]
Remove first: [one source of friction]
Then: [next action or focus block]
```

## Daily triage

```markdown
Today's outcome: [most important result]

| Order | Task | Done condition | Consequence | ABCDE | Decision |
| --- | --- | --- | --- | --- | --- |
| 1 | [frog] | [condition] | [why it matters] | A1 | Do first |
| 2 | [task] | [condition] | [impact] | A2/B | Do next or schedule |
| 3 | [task] | [condition] | [impact] | B/C/D/E | Defer, delegate, or eliminate |

Start now: [first action]
Focus block: [duration] to produce [artifact]
Communication window: [proposed time]
```

## Frog selection check

Use qualitative ratings unless the user asks for numerical scoring.

```markdown
| Candidate | Consequence | Goal contribution | Unblocks work | Only I can do | Deadline reality |
| --- | --- | --- | --- | --- | --- |
| [task] | High/Med/Low | High/Med/Low | High/Med/Low | Yes/No | [date/context] |
```

Select the task with the strongest combined case. Explain the decision in one sentence.
